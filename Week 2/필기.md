# Software Development Cycle
Analysis;
Establish the Goal of the Program.
Define Input and Output.

Design;
Algorithm & ADT Specification.

Implementation;
Coding & Debugging.

Verification & Maintenance;
Verify Correctness via Testing.
Expand its Utility and Stability by Maintenance.

# Algorithm
Requirments;
Input, Output, Definiteness, Finiteness, Effectiveness.

# ATD (Abstract Data Type)
We don't want to perform different operation based on how the underlying data structure is constructed.

# Recursive
Pros : Natural to represent algorithms & Clarity to reduce the time needed to debug
Cons : More memory usage & Time-consuming than its iterative counterpart

# Stack
LIFO : Last In First Out
Basic Operations : push, pop

# Queue
FIFO : First In First Out
Basic Operations : enqueue, dequeue

# Performance Evaluation
Machine dependent mesurment is unstable. 
Machine independent mesurments (=Asymptotic Analysis) : Space & Time Complexity

# Space Complexity
Fixed Space : Doesn't affected by input.
Variable Space : Affected by program's input size.

# Time Complexity
Type : n!, 2^x, x^2, nlogn, n, logn, 1
Big O : Upper Bound
Big-𝚯 : Lower and Upper Bound
Big-𝛀 : Lower Bound
